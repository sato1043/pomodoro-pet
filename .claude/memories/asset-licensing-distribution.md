# 素材ライセンスと配布方式

## 概要

本プロジェクトはElectronアプリとして配布する。ソースコードの著作権は開発者が保持し、GitHubで公開する。一方、購入素材（音声・効果音・画像・3Dモデル）は開発者に著作権がないため、GitHubリポジトリには含められない。この文書はその前提で配布を成立させるための調査結果と方針をまとめる。

**注意: 本文書は法的助言ではない。具体的な判断は弁護士に確認すること。**

## 前提条件

- ソースコード: 開発者が著作権を保持。GitHubで公開する
- 購入素材: 第三者が著作権を保持。ライセンスに基づいて使用している
- 配布形態: ビルド済みElectronバイナリ（Windows NSIS/展開済みディレクトリ）
- Electronの技術的特性: `app.asar`は`asar extract`で容易に展開でき、素材は元品質で抽出可能

## 1. 素材ライセンスの類型と配布可否

| ライセンス類型 | GitHub公開（素材単体） | バイナリ同梱（ビルド済み配布） | 素材単体の再配布 |
|---|---|---|---|
| Royalty-Free（組込み許可） | 不可 | 通常は可 | 不可 |
| Extended License | 不可 | 通常は可（条件付き） | 不可 |
| Editorial Use Only | 不可 | 不可（商用不可） | 不可 |
| 自作 / CC0 / パブリックドメイン | 可 | 可 | 可 |

### 確認すべきライセンス条項のキーワード

- `redistribution` — 再配布の可否と条件
- `end product` — 最終製品の定義（素材が不可分に組み込まれているか）
- `sub-licensing` — 第三者へのライセンス付与の可否
- `derivative works` — 二次的著作物の扱い
- `number of installations / users` — インストール数やユーザー数の制限

## 2. Electronアプリ（asar）の素材保護

### 現実

- `app.asar`は`asar extract`コマンドで即座に展開可能
- 音声・画像・3Dモデルはほぼ元品質で抽出できる
- 「最終製品への不可分な組み込み」とは言いがたい状態

### 業界通念

- 多くのElectronアプリ・ゲーム（RPGツクール製等含む）が同様の状態で配布されている
- 多くのRoyalty-Freeライセンスは「最終製品の一部としての配布」を許可しており、ユーザーが技術的に抽出*できる*こと自体はライセンス違反とされないのが一般的理解
- 素材の抽出・再利用は**抽出した側のライセンス違反**となる

### カジュアル抽出の抑止策（任意）

完全な保護は不可能だが、カジュアルな抽出を抑止する手段はある：

- asarアーカイブの使用（electron-builderのデフォルトで有効）
- 素材ファイル名の難読化
- 独自フォーマットへの変換（音声→独自コンテナ等）
- カスタム暗号化+実行時復号

これらは技術的抑止であり、法的保護ではない。

## 3. GitHubリポジトリの構成方式

### 方式A: プライベートストレージ + ダウンロードスクリプト（推奨）

```
assets/              ← .gitignoreに追加
assets/README.md     ← 必要な素材の一覧・入手先・配置方法を記載
scripts/
  download-assets.sh ← プライベートS3/GCS等からダウンロード（認証付き）
```

- 利点: CIからの自動ビルドが容易。素材管理が明確
- 欠点: ストレージコスト。認証管理が必要
- 適用例: S3, Google Cloud Storage, Azure Blob Storage

### 方式B: Git Submodule（プライベートリポジトリ）

```
assets/  ← private repoへのsubmodule
```

- 利点: Gitの仕組みで自然に管理できる。バージョン管理が容易
- 欠点: コントリビューターがsubmoduleをcloneできない。CI設定にdeploy keyが必要

### 方式C: リリースバイナリのみ配布

- GitHub Releasesにビルド済みバイナリをアップロード
- ソースコードは公開するが、アセットなしではビルド不完全であることをREADMEに明記
- プレースホルダーアセットを用意してビルド自体は通るようにする

### 共通の実装要件

どの方式でも以下が必要：

- **プレースホルダーアセット**: 素材がなくてもビルド・起動が通るダミーファイル（無音mp3、1x1 png等）
- **アセットマニフェスト**: 必要な素材のファイル名・フォーマット・入手先の一覧
- **ビルドスクリプトの条件分岐**: 本番素材がなければプレースホルダーでフォールバック

## 4. 実務上のチェックリスト

### ライセンス確認

- [ ] 全購入素材のライセンス条項を一覧化する
- [ ] 各素材について「バイナリ同梱配布」が許可されているか確認する
- [ ] 不明確な場合は素材提供元に問い合わせる（「Electronアプリとして配布し、技術的にはasar展開で素材抽出が可能だが、最終製品としての配布は許可されるか」）
- [ ] ライセンスで必要なクレジット表記を確認する

### アプリへの表記

- [ ] `THIRD_PARTY_LICENSES`ファイルに素材のクレジットとライセンスを記載する
- [ ] EULAまたはREADMEに「含まれる素材の抽出・再利用は禁止」と明記する
- [ ] About画面にクレジット表記を表示する（ライセンスで要求される場合）

### リポジトリ構成

- [ ] `assets/`を`.gitignore`に追加する
- [ ] アセット取得方法（方式A/B/C）を決定・実装する
- [ ] プレースホルダーアセットを用意し、素材なしでもビルドが通ることを確認する
- [ ] READMEに素材の入手・配置方法を記載する

### フリー素材への置換検討

- [ ] 法的リスクを根本的に排除するため、可能な素材をCC0/パブリックドメインに置換できるか検討する
- [ ] 置換候補の素材ソース: Freesound (音声), OpenGameArt (画像/3D), Poly Pizza (3D), Pixabay (音声/画像)
