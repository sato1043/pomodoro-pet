# 有料配布方式の設計 — itch.io「Direct to you」モード

## 配布方式概要

| 項目             | 決定 |
|------------------|---|
| プラットフォーム | itch.io |
| 支払いモード     | Direct to you |
| 価格             | $4.99 USD（itch.io手数料10%） |
| 返金             | 受け付けない / 販売ページ・EULA・購入前画面で目立つように明記する |
| 準拠法           | 日本法 |
| インストーラー   | ワンクリック（`oneClick: true`）。EULA同意は購入ページで担保 |
| リポジトリ構成   | submodule方式（コード本体はpublic、購入素材はprivateサブモジュールに分離。CI/CDの複雑さは許容） |


## itch.io「Direct to you」の仕組み

- 購入者の支払いがitch.ioを経由せず、販売者のアカウントに直接入金される方式
- 販売者はPayPalアカウントとStripeアカウントをそれぞれ接続できる
    - PayPalのみ接続 → 購入者はPayPalのみ利用可能
    - Stripeも接続 → 購入者はクレジットカードも利用可能
    - 両方接続すれば購入者の選択肢が広がる
- 「Collected by itch.io, paid later」（itch.ioが決済代行、月次支払い）との違い:
    - 入金が即時（月次支払い待ち不要）
    - 購入者の支払いが販売者に直接入金。支払い遅延リスク回避
    - EU VAT処理は自己責任（itch.ioが代行しない）
    - Stripe接続は販売者自身のアカウントが必要

### itch.ioアカウント準備

- Tax Interview提出（W-8BENをオンライン提出）
- W-8BENにマイナンバー記入（日米租税条約適用で米国源泉徴収0%へ軽減）
- PayPalアカウント接続（10万円超の受取にはマイナンバー+本人確認必要）

注意点:
- FTIN（マイナンバー）を入力しないと日米租税条約の適用を主張できない。
  未入力だと米国源泉徴収が30%かかる可能性がある
- itch.io の Tax Interview は裏で IRS（米国内国歳入庁）のシステムと連携している。
  情報は正確に入力すること
- PayPal の接続（項目3）は Tax Interview と独立した設定。
  Tax Interview を先に済ませるのが望ましい

### itch.io Tax Interview（W-8BEN提出）の操作ガイド

前提条件:
- itch.io アカウントが作成済みであること
- マイナンバー（個人番号）が手元にあること
- 本名（パスポート表記のローマ字）が確認できること

手順:

1. Tax Interview ページへアクセス
    1. itch.io にログイン
    2. 右上のアカウントメニュー → Dashboard を開く
    3. 左メニューの Settings をクリック
    4. Tax information セクションを探す → "Begin tax interview" ボタンをクリック

2. 基本情報の入力
    1. Type of beneficial owner: Individual を選択
    2. Country of citizenship: Japan を選択
    3. Are you a U.S. citizen or resident?: No を選択

    → W-8BEN フォームに誘導される（米国非居住の個人向け）

3. W-8BEN フォームの記入

    ┌──────────────────────┬──────────────────────────────────────────────────────────────────────────────────┐
    │      フィールド      │                                     入力内容                                     │
    ├──────────────────────┼──────────────────────────────────────────────────────────────────────────────────┤
    │ Name of beneficial   │ パスポート表記のローマ字氏名（例: Taro Yamada）                                  │
    │ owner                │                                                                                  │
    ├──────────────────────┼──────────────────────────────────────────────────────────────────────────────────┤
    │ Country of           │ Japan                                                                            │
    │ citizenship          │                                                                                  │
    ├──────────────────────┼──────────────────────────────────────────────────────────────────────────────────┤
    │ Permanent residence  │ 日本の自宅住所を英語表記で入力                                                   │
    │ address              │                                                                                  │
    ├──────────────────────┼──────────────────────────────────────────────────────────────────────────────────┤
    │ Foreign tax          │                                                                                  │
    │ identifying number   │ マイナンバー（12桁）を入力 ← これが重要                                          │
    │ (FTIN)               │                                                                                  │
    ├──────────────────────┼──────────────────────────────────────────────────────────────────────────────────┤
    │ Tax treaty benefits  │ チェックを入れる                                                                 │
    ├──────────────────────┼──────────────────────────────────────────────────────────────────────────────────┤
    │ Country              │ Japan を選択                                                                     │
    ├──────────────────────┼──────────────────────────────────────────────────────────────────────────────────┤
    │ Article and          │ itch.io側で自動選択される場合が多い。手動の場合は Article 12（Royalties）        │
    │ paragraph            │                                                                                  │
    ├──────────────────────┼──────────────────────────────────────────────────────────────────────────────────┤
    │ Rate of withholding  │ 0%                                                                               │
    ├──────────────────────┼──────────────────────────────────────────────────────────────────────────────────┤
    │ Conditions:          │ 「The beneficial owner is a resident of Japan within the meaning of the income   │
    │                      │ tax treaty between the United States and that country」の趣旨                    │
    └──────────────────────┴──────────────────────────────────────────────────────────────────────────────────┘

4. 電子署名と提出
    1. 署名欄にフルネーム（ローマ字、上記と同一）を入力
    2. "I certify that I have the capacity to sign..." のチェックボックスにチェック
    3. Submit をクリック

5. 完了確認
    - Tax information セクションに "W-8BEN on file" または同等のステータスが表示されれば完了
    - W-8BEN の有効期限は 提出日の属する暦年の最終日から3年間（例: 2026年提出 → 2029年12月31日まで有効）

### Tax information セクションがない                                                                              

販売者（Seller）としてのセットアップがまだ完了していないと、
Tax information セクションが表示されない。

itch.io では有料販売の設定を行った後に Tax Interview が出現する。
Projectを Draft で作ることで表示されるようになる。

  1. Dashboard → Settings を開いた状態で、どのようなセクション/タブが見えているか
  2. "Seller settings" または "Payment" というセクションやタブがあるか
  3. すでにプロジェクト（ゲームページ）を1つでも作成しているか

もしまだであれば、先にやるべき手順は以下の可能性が高い。

  1. Dashboard → Create new project でゲームページを作成（ドラフトでよい）
  2. ゲームページの Pricing で "$X.XX or above"（有料）を選択
  3. Payment mode で "Direct to you" を選択
  4. この時点で PayPal/Stripe の接続を求められる → PayPal アカウントを接続
  5. PayPal 接続後に Tax Interview が表示される


### 手数料の内訳（$4.99販売時の試算）

#### PayPal経由

| 項目           | 金額                        |
|----------------|-----------------------------|
| itch.io手数料  | $0.50 （10%）               |
| 決済手数料     | $0.44 （2.9%+$0.30）        |
| 通貨換算       | $0.14 （3.5%想定）          |
| **手取り**     | **$3.91 （約78%）**         |

#### Stripe経由（日本アカウント）

| 項目           | 金額                             |
|----------------|----------------------------------|
| itch.io手数料  | $0.50 （10%）                    |
| 決済手数料     | $0.18〜$0.22 （国内3.6%/海外4.4%）|
| 通貨換算       | $0.09 （2%想定）                 |
| **手取り**     | **$4.18〜$4.21 （約84%）**       |

※Stripeは固定費（$0.30）がなく、少額取引では手取りが多い

※米国購入者の場合、日米租税条約適用で源泉徴収0%を想定

※日本側の税務
- 確定申告（継続販売なら事業所得、副業的なら雑所得）
- 外国税額控除（米国源泉徴収があった場合）
- 詳細は税理士に相談

※itch.ioの「Direct to you」モードでのEU VAT対応
- itch.io の「Direct to you」モードでは、itch.io は決済の仲介ではなく PayPal 直接入金になる。この場合、販売者（開発者）が EU の顧客に対して VAT（付加価値税）の徴収・申告義務を負う可能性がある。
- EU VAT の概要:
    - EU 居住者へのデジタル商品販売は、購入者の国の VAT 率（15-27%）が適用される
    - 年間売上が一定額以下なら免除される国もある
    - EU VAT の申告には VAT MOSS（Mini One Stop Shop）への登録が必要
- 実際のリスク:
    - 個人開発者が月数件の販売で EU VAT 当局に追及される可能性は極めて低い
    - itch.io の「Collected by itch.io, paid later」モードなら itch.io が VAT 処理を代行してくれる
    - itch.io の「Direct to you」では自己責任
    - PayPal は購入者の国を開発者に通知しないため、EU 購入者の特定自体が困難
- 現時点では対応不要。売上が大きくなった段階で税理士に相談。
    - または「Collected by itch.io, paid later」モードに切り替えればitch.ioがVAT処理を代行する（入金が月次になる）

- 日本の法律で、消費者向けにネット販売する事業者に一定の情報表示を義務づける制度。

### SmartScreen のレピュテーション（署名実績の蓄積）

Windows SmartScreen は OV/IV 証明書で署名されたアプリに対して、初回から無条件に信頼するわけではない。
証明書ごとに「レピュテーション（評判）」を蓄積し、十分な実績が溜まるまで警告が出る場合がある。

| 状態 | SmartScreen の挙動 |
|------|-------------------|
| 署名なし | 毎回「不明な発行元」警告 |
| OV証明書（実績なし） | しばらく警告が出る可能性あり |
| OV証明書（実績蓄積後） | 警告なし |
| EV証明書 | 初日から警告なし |

- 警告が出るタイミング: インストーラー（.exe）の実行時。アップデート時にも出る可能性がある
  - electron-updater による自動アップデートはバックグラウンドで処理されるため、ユーザーが警告を見るのは主に手動インストール時
- 「何件で信頼されるか」の閾値は Microsoft が非公開
- 数百〜数千ダウンロード程度で解消されるという報告が多い
- OV証明書でも署名なしより警告の文面が穏やかになり、解消も早い
- 販売規模を考えると警告が出つづける可能性は高いが OV証明書でも署名なしよりははるかにマシ
- 対応: itch.io 販売ページに SmartScreen 警告の回避手順を注記する

### SmartScreen 警告の回避手順（販売ページ・READMEに掲載する）

インストール時に「Windows によって PC が保護されました」と表示される場合がある。
以下の手順で実行できる。

1. 「Windows によって PC が保護されました」ダイアログが表示される
2. **「詳細情報」** をクリックする（ダイアログ下部の小さなリンク）
3. 発行元の情報とファイル名が表示される
4. **「実行」** ボタンをクリックする

この警告はソフトウェアの署名実績が十分に蓄積されていない場合に表示されるもので、
ソフトウェア自体に問題があるわけではない。

## 自動アップデート + ライセンス管理

### アップデート配信方式
- GitHub Releases（public）+ electron-updater
- `app-update.yml` がビルド時に自動生成され、GitHub Releases のURLを参照
- `package.json` の `build.publish` にGitHub provider設定済み

### ハートビートAPI
- GCP Cloud Function（2nd gen, Node.js 22）+ Firestore
- POST /api/heartbeat: デバイス登録状態確認・トライアル管理・JWT発行
- POST /api/register: itch.io download keyによる登録
- 詳細: [gcp-update-server.md](gcp-update-server.md)

### トライアル
- 30日間、サーバー側（Firestore）で管理
- トライアル中は全機能利用可能
- トライアル期間後: 基本タイマー+キャラクター表示は継続。設定・統計・ふれあい等は制限

### 登録
- itch.io download keyをアプリに入力して登録
- サーバーがitch.io APIでkey検証 → Firestoreに記録 → JWT発行
- 台数制限: download key 1つにつきN台まで（N は config/current で設定）

### JWT（オフライン認証）
- RS256署名（サーバー秘密鍵で署名、アプリ公開鍵で検証）
- ペイロード: { deviceId, keyHint, iat, exp }
- 有効期限30日（ハートビート成功時にローリング更新）
- 24時間以内のJWTはハートビートをスキップ
